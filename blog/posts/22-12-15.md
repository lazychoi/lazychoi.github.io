## 자바

XML을 공부해야 XML을 기반으로 만들어진 JSON, GML(Geometric Markup Langugae) 등을 이해할 수 있음.

- 배열과 구조체를 합친 것이 파이썬의 list
- 배열은 연속적인 메모리 공간에 할당됨
- int a[5] -> 4바이트 크기의 방 5개 할당: a[0] ~ a[4]
- 배열은 같은 자료형만 쓸 수 있음.
- 여러 자료형을 묶어서 쓸 수 있도록 한 것이 **구조체**(C에서 유래) eg. 학생-번호, 이름, 나이, 주소 등

```
Struct
{
	int number
	char name[5]
	int age
	char address[20]
}Student
```

- 구조체 -> **클래스**(구조체+함수)
- 데이터 관리
  - 디스트 : 
    - 파일처리(동시 사용 불가능) -> 데이터베이스(DBMS가 여러 프로그램의 파일 접근을 스케줄링)
	- **디스트 IO 최소화**
  - 메모리 : 
    - 자료구조 : 메모리에 데이터 저장. int, char, ... -> 배열, 구조체, .. -> 스택, 큐, ..
	- 알고리즘 : 메모리에서 효율적 데이터 처리. **비교 횟수 최소화**. 정렬, 검색, ...

변수

- 객체지향 아닌 경우
  - 지역 변수: 블록 안에서 선언. 블록 안에서만 사용 가능.
  - 전역 변수: 블록 밖에서 선언
- 객체지향인 경우
  - 지역/전역
  - 클래스 변수: 첫번째 객체가 만들어질 때 생성된 변수를 다른 객체가 같이 사용할 때의 변수. eg. 클래스에서 static으로 선언.
  - 객체 변수: 각 객체가 생성될 때의 변수

생성자 함수, 소멸자 함수

- 생성자 함수: 객체가 생성될 때 자동으로 호출됨
- 소멸자 함수: 할당한 메모리 해제.
